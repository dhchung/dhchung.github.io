Bundle Adjustment

15 Nov 2021 in Research Log

Bundle adjustment from scratch.

Bundle adjustment. 하 진짜. VO 한다는 사람이 이제서야 제대로 BA에 대해 공부하기 시작한 웃지 못할 상황이다. 사실 Bundle Adjustment (와 딱 1년전이네 ㄷㄷ)에서 좀 다뤄볼려고 했는데, 그땐 거의 Cyrill 교수님 강의 베낀 수준이라 완벽히 이해하진 못한거 같다. 이번에 직접 implementation까지 하면서 좀 제대로 이해해보려고 한다.

BA에 대해서는 사실 석사 때부터 많이 들어오긴 했는데, 제대로 공부할 기회가(라고 쓰고 생각이 라고 읽늗다) 없었다. 걍 뭐 optimization 하나보다 생각했는데, 이번에 좀 제대로 정립하고자 여기저기서 긁어서 공부하며 끄적인다.

BA의 목적

  • BA는 기존 DLT나 PnP 알고리즘과 같이 한번에 unknown paramter들을 계산하는 방식이 아닌 iterative한 방법으로 optimization하는 과정이다.
  • 여기서 optimization 대상은 rotation, translation, 3d point들이며 적용 목적에 따라 대상이 달라질 수 있다.
  • Iterative procedure이기에 Initial guess가 중요하다.
  • 완벽히 이해는 못했지만, control point들이나 다른 factor들과 같이 융합이 가능하다 (odometry 정보라든지)

Reprojection Error

모든 Optimization problem이 다 그렇듯이 제대로 된 cost function (DL에서의 loss와 비슷한 개념)이 있어야 좋은 결과가 나온다.

BA에서는 reprojection error를 사용한다. 이를 설명하기 전에 추정된 3D point cloud와 image point와의 관계는 다음과 같다.

\[x_{ij}^h + v_{x_{ij}}^h=\lambda_{ij}P(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h\]
  • \(x_{ij}^h\)는 원래 이미지의 2D point location으로 homogeneous coordinate \((x_ij, y_ij, 1)^\top\)으로 표현 (i번째 point의 j번째 image plane의 2d coordinate)
    • \({x_{ij}}\)는 image space에서의 2d point coordinate
  • \(v_{x_{ij}}^h\)는 reprojection error
  • \(\lambda_{ij}\)는 scale factor
  • \(P(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j)\)는 camera projection matrix
  • \(\mathbf{X}_i^h\)는 3차원의 point cloud로 homogeneous coordinate \((X_i, Y_i, Z_i, 1)^\top\)으로 표현 (i번째 point)
  • \(\mathbf{r}_j\)와 \(\mathbf{t}_j\)는 각각 WC기준 카메라의 rotation 및 translation를 나타내는 vector
    • \(\mathbf{r}\)는 다양하게 표현 가능 (Euler angle, rotation vector, quaternion, …)
    • \(\mathbf{r}\)에 대해 여기선 rotation vector \((\omega_x, \omega_y, \omega_z)^\top\)을 사용
    • \(R(\mathbf{r}) = I + \frac{sin(\theta)}{\theta}\Omega + \frac{1-cos(\theta)}{\theta^2}\Omega^2\) (Rodrigues formula)
      • \(\theta\) \(= \lVert(\mathbf{r})\rVert\)
      • \(\Omega\) \(= \mathbf{r}_{[\times]} = \begin{bmatrix} 0 && -\omega_z && \omega_y \\ \omega_z && 0 && -\omega_x \\ -\omega_y && \omega_x && 0 \end{bmatrix}\)
      • 역으로 구하는 방법

여기서 \(P(\mathbf{p},\mathbf{r},\mathbf{t})=\) \(K(\mathbf{p})[R(\mathbf{r})~\vert~\mathbf{t}]\) 로 intrinsic parameter \(\mathbf{p}\)에 대한 함수인 camera matrix \(K\)와 카메라의 위치 및 자세 정보 \(\mathbf{t}, \mathbf{r}\)로 구성된다.

대부분의 computer vision 강의에선 카메라 자세 및 위치를 반대로 표현했는데, 회전이랑 변위를 각각 다르게 표현해서 헷갈려서 그냥 로보틱스에서 주로 쓰는 방법 기준으로 작성했다. (Essential & Fundamental Matrix의 Rotataion matrix & translation 참고) 여기서 주의해야 할 점은, 여기서의 R과 t는 카메라 좌표계에서 바라본 WC의 rotation 및 translation이라는 점이다. 카메라의 회전 및 자세를 구하기 위해선 구해진 R과 t에 대한 inverse를 취해야 된다.

\[[R_{cam} ~ \vert ~ \mathbf{t}_{cam}] = [R^\top ~ \vert -R^\top \mathbf{t}]\]

Reprojection error를 minimise하기 위해 이의 squared sum을 구한다.

\[(\mathbf{r}_{j=1,...,m}, \mathbf{t}_{j=1,...,m}, \mathbf{X}_{i=1,...n})=\text{argmin}_{\mathbf{r}, \mathbf{t}, \mathbf{X}}\sum_{i=1}^{n}\sum_{j=1}^{m}{\gamma_{ij} \lVert \lambda_{ij}P(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h - x_{ij}^h \rVert^2 }\]

여기서 i번째 point가 j번째 image에 속해있으면 \(\gamma_{ij}=1\) otherwise, 0.

이는 결국 WC에서 표현된 3차원 point cloud를 카메라에 projection 시켰을 때, 처음의 image point과의 L2 norm을 나타낸다. (Cyrill Lecture에선 Huber norm이 쓰인다.)

Scale factor \(\lambda_{ij}\)가 차지하는 unknown수의 비중이 굉장히 높으므로 이를 제거하기 위해 depth축으로 나눠준다.

\[x_{ij} + v_{x_{ij}}=P_{(1:2,:)}(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h / P_{(3,:)}(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h\]

Scale factor를 제거한 cost function은 다음과 같이 된다.

\[A(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j, \mathbf{X}_i)=P_{(1:2,:)}(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h / P_{(3,:)}(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h\] \[C(x_{ij}, \mathbf{p}, \mathbf{r}_j, \mathbf{t}_j, \mathbf{X}_i) =\sum_{i=1}^{n}\sum_{j=1}^{m}{\gamma_{ij} \lVert A(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j, \mathbf{X}_i - x_{ij}') \rVert^2 }\] \[(\mathbf{r}_{j=1,...,m}, \mathbf{t}_{j=1,...,m}, \mathbf{X}_{i=1,...n})=\text{argmin}_{\mathbf{r}_j, \mathbf{t}_j, \mathbf{X}_i} C(x_{ij}, \mathbf{p}, \mathbf{r}_j, \mathbf{t}_j, \mathbf{X}_i)\]

결국 이 cost function이 최소가 되는 \(\mathbf{r}, \mathbf{t}, \mathbf{X}\)들을 찾는 문제가 된다.

Jacobian of Reprojection Error

대부분의 optimization problem이 그렇듯, 그리고 DL에서도 사용되는 gradient decent 방법으로 최소값이 되는 camera pose & point cloud를 찾아낸다.

일단 어떤 값에 대해 이 cost function을 편미분해보자.

\[\frac{\partial(C)}{\partial(\mathbf{a})} = 2\sum_{i=1}^{n} \sum_{j=1}^{m} \gamma_{ij} \lVert A(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j, \mathbf{X}_i) - x_{ij}' \rVert \frac{\partial A(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j, \mathbf{X}_i)}{\partial \mathbf{a}}\] \[=2\sum_{i=1}^{n} \sum_{j=1}^{m} \gamma_{ij} \lVert A(\mathbf{p},\mathbf{r}_j,\mathbf{t}_j, \mathbf{X}_i) - x_{ij}' \rVert \left( \frac{\partial A}{\partial \mathbf{p}}\frac{\partial \mathbf{p}}{\partial \mathbf{a}}+ \frac{\partial A}{\partial \mathbf{r}_j}\frac{\partial \mathbf{r}_j}{\partial \mathbf{a}} + \frac{\partial A}{\partial \mathbf{t}_j}\frac{\partial \mathbf{t}_j}{\partial \mathbf{a}}+ \frac{\partial A}{\partial \mathbf{X}_i}\frac{\partial \mathbf{X}_i}{\partial \mathbf{a}} \right)\]

사실 엄밀히 말해 camera intrinsic parameter인 \(\mathbf{p}\)도 영상마다 달라질 수 있어(기온차에 의한 렌즈 변형, 이미지별 다른 카메라 및 렌즈) \(\mathbf{p}_i\)로 해주는게 맞으며, intrinsic parameter 또한 이런 방법으로 구할 수 있지만, 여기서는 그냥 알고 있는 constant라 하자. (practical하게 생각한 경우 보통 camera calibration을 먼저 수행하고 로봇을 움직이니까) 그럼 위 식은 다음과 같이 된다.

\[\frac{\partial(C)}{\partial(\mathbf{a})} = 2\sum_{i=1}^{n} \sum_{j=1}^{m} \gamma_{ij} \lVert A(\mathbf{r}_j,\mathbf{t}_j, \mathbf{X}_i) - x_{ij}' \rVert \left(\frac{\partial A }{\partial \mathbf{r}_j}\frac{\partial \mathbf{r}_j}{\partial \mathbf{a}} + \frac{\partial A}{\partial \mathbf{t}_j}\frac{\partial \mathbf{t}_j}{\partial \mathbf{a}} + \frac{\partial A}{\partial \mathbf{X}_i}\frac{\partial \mathbf{X}_i}{\partial \mathbf{a}} \right)\]

여기서 각 \(\mathbf{r}_j, \mathbf{t}_j, \mathbf{X}_i\)에 대해 \(A\)를 편미분 해야 되는데, scaling factor를 없애기 위해 1,2번째 row를 3번째 row로 나눠 미분하기 좀 불편하다. 따라서 이를 나누지 않은 식에 대해 먼저 미분한 다음 chain rule을 통해 다음과 같이 해결한다.

\[\begin{bmatrix}u\\ v\\ w\end{bmatrix} = B(\mathbf{r}_j, \mathbf{t}_j, \mathbf{X}_i) = \lambda_{ij}P(\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h\]

라 하면 우리가 적용한 식은

\[\begin{bmatrix}u/w\\ v/w\end{bmatrix} = A(\mathbf{r}_j, \mathbf{t}_j, \mathbf{X}_i) = (P_{(1:2,:)}(\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h)/((P_{(3,:)}(\mathbf{r}_j,\mathbf{t}_j)\mathbf{X}_i^h)\]

이며, A에 대해 b로 편미분 한 값은 다음과 같이 표현된다.

\[\frac{\partial A}{\partial \mathbf{b}} = \frac{\partial A}{\partial B}\frac{\partial B}{\partial \mathbf{b}} = \begin{bmatrix} 1/w && 0 && -u/{w^2} \\ 0 && 1/w && -v/{w^2} \end{bmatrix}\frac{\partial B}{\partial \mathbf{b}}\]

원래 식은 다음과 같이 된다.

\[\frac{\partial(C)}{\partial(\mathbf{a})} = 2\sum_{i=1}^{n} \sum_{j=1}^{m} \gamma_{ij} \lVert A(\mathbf{r}_j,\mathbf{t}_j, \mathbf{X}_i) - x_{ij}' \rVert \begin{bmatrix} 1/w && 0 && -u/{w^2} \\ 0 && 1/w && -v/{w^2} \end{bmatrix} \left(\frac{\partial B }{\partial \mathbf{r}_j}\frac{\partial \mathbf{r}_j}{\partial \mathbf{a}} + \frac{\partial B}{\partial \mathbf{t}_j}\frac{\partial \mathbf{t}_j}{\partial \mathbf{a}} + \frac{\partial B}{\partial \mathbf{X}_i}\frac{\partial \mathbf{X}_i}{\partial \mathbf{a}} \right)\]

로 표현이 된다.

\(B\)는 \(K[R(\mathbf{r})_j~\vert~\mathbf{t}_j]\mathbf{X}_i^h\)이므로 각각에 대해 편미분 해줘야 한다.

이를 다시 쓰면

\(B=\) \(K(R(\mathbf{r}_j)\mathbf{X}_i + \mathbf{t}_j)\) 가 된다. (여기서 주목할 것은 \(\mathbf{X}_i^h\) 에서 \(\mathbf{X}_i\)로 변경된 점이다. Homography coordinate에서 Euclidean Coordinate로)

이제 \(\mathbf{a}\)를 우리가 구하고자 하는 unknown에 대해 바꾸면서 계산하면 된다.

\(\mathbf{a} = \mathbf{X}_i\)에 대해

\[\frac{\partial B}{\partial \mathbf{X}_i} = \frac{\partial (KR(\mathbf{r}_j)\mathbf{X}_i + \mathbf{t}_j)}{\partial \mathbf{X}_i} = KR(\mathbf{r}_j)\]

\(\mathbf{a} = \mathbf{t}_j\)에 대해

\[\frac{\partial B}{\partial \mathbf{t}_j} = \frac{\partial (KR(\mathbf{r}_j)\mathbf{X}_i + \mathbf{t}_j)}{\partial \mathbf{t}_j} = K\]

\(\mathbf{a} = \mathbf{r}_j\)에 대해

\[\frac{\partial B}{\partial \mathbf{r}_j} = \frac{\partial (KR(\mathbf{r}_j)\mathbf{X}_i + \mathbf{t}_j)}{\partial \mathbf{r}_j} = \frac{\partial (KR(\mathbf{r}_j)\mathbf{X}_i)}{\partial \mathbf{r}_j} = K\frac{\partial (R(\mathbf{r}_j))}{\partial \mathbf{r}_j}\mathbf{X}_i\]

이게 사실 좀 복잡한데, 암튼 Rodrigues formula를 참고하고 여차저차 풀면 된다. 자세한 내용은 진용이형 블로그에서 참고

\[\frac{\partial R}{\partial \omega} = \frac{\partial}{\partial \omega} \left( I + \frac{sin(\theta)}{\theta}\Omega + \frac{1-cos(\theta)}{\theta}\Omega^2 \right)\] \[\frac{\partial}{\partial \omega}\left( \frac{sin(\theta)}{\theta} \right)\Omega + \frac{sin(\theta)}{\theta} \frac{\partial \Omega}{\partial \omega}+ \frac{\partial}{\partial \omega}\left( \frac{1 - cos(\theta)}{\theta^2} \right)\Omega^2 + 2\frac{1 - cos(\theta)}{\theta^2} \frac{\partial \Omega}{\partial \omega}\]

복잡해 보이지만, 복잡하게 풀다 보면 복잡하게 패턴이 나온다 (읭?). 모든 풀이 내용은 위에 블로그를 참고하였다.

\[\frac{\partial \theta}{\partial \omega_i} = \frac{1}{2} (\omega_x^2 + \omega_y^2 + \omega_z^2)^{\frac{1}{2}} (2\omega_i) = \frac{\omega_i}{2}\] \[\frac{\partial}{\partial \omega_i}(\frac{sin(\theta)}{\theta}) = \frac{(cos(\theta)\theta - sin(\theta))\omega_i}{\theta^3}\] \[\frac{\partial}{\partial \omega_i}(\frac{1-cos(\theta)}{\theta^2}) = \frac{(sin(\theta)\theta - 2 + 2cos(\theta))\omega_i}{\theta^4}\]

where \(i=\{x,y,z\}\)

\(\theta\)가 \(\omega_i\)의 순서에 invariant하기에 여기까지는 다 같은 형태다.

결국 달라지는 것은 \(\frac{\partial \Omega}{\partial \omega_i}\)이며 이는 다음과 같다.

\[\frac{\partial \Omega}{\partial \omega_x}=\begin{bmatrix}0 && 0 && 0\\ 0 && 0 && -1 \\ 0 && 1 && 0\end{bmatrix}\] \[\frac{\partial \Omega}{\partial \omega_y}=\begin{bmatrix}0 && 0 && 0\\ 1 && 0 && 0 \\ -1 && 0 && 0\end{bmatrix}\] \[\frac{\partial \Omega}{\partial \omega_z}=\begin{bmatrix}0 && -1 && 0\\ 1 && 0 && 0 \\ 0 && 0 && 0\end{bmatrix}\]

최종적으로 이들을 넣어 Jacobian을 계산하면 된다.
