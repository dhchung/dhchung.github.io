I need to study about threading!!!

10 Dec 2021 in programming

Threading

SLAM 공부를 하며 직접 구현을 하면서 threading을 잘 모르니 여러 어려움이 있어 내용을 정리할 겸 실습을 진행해봤다.

일단 테스트를 위해 현재 구현하고자 하는 프로그램을 대상으로 진행해봤다.

기본적으로 sensor sequence에 대해 재생하는 프로그램을 만들어야 되는데, 여기서는 일단 영상에 대해서만 진행하고자 한다. 시나리오는 다음과 같다. 영상을 불러오고 처리하는 과정에서 불러오는 부분과 처리하는 부분을 다른 thread로 만드는 과정이다. 이부분을 한개의 thread만 사용하여 처리하면 다음과 같다.

여기서 image loading에 \(t_1\)의 시간을, image processing에 \(t_2\)의 시간이 요구된다면, 전체 process에 \(t_1 + t_2\)의 시간이 소요된다.

각각을 thread로 처리하면 다음과 같다.

이런 구조로 진행한다면 image processing 하는 시간에 image loading이 진행이 되어 시간은 \(t_1 + t_2\)에서 \(t_2\)로 짧아진다.

여기서 Keyword는 각각 thread가 input이 없을 때 쓸데 없이 돌아가는 것을 방지하기 위한 condition_variable과 공통으로 사용되는 variable에 대해 thread가 동시에 접근하는 것을 막는 mutex이다.

Mutex란 mutual exclusive의 준말로 각기 다른 thread에서 동시에 같은 variable에 대한 access를 방지하기 위해 사용된다.

c++ unix에서 프로그래밍 할때는

#include <thread>
#include <mutex>
#include <condition_variable>

로 include해주면 된다.

QT는 자신만의 thread, mutex, condition_variable을 사용하는데, 이는

#include <QThread>
#include <QMutex>
#include <QWaitCondition>

을 사용하는 모양이다.

일단 이 포스트는 나중에 다시 써야겠다. 구현은 다 했는데, 너무 바빠서 @_@
