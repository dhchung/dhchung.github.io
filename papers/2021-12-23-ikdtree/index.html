ikd-Tree (2021 arxiv)

23 Dec 2021 in Paper Review

ikd-Tre: An Inceremtnal K-D Tree for Robotic Applications

저자는 홍콩대의 MaRS Lab 연구진들이다. Fast-LIO, Fast-LIO2를 보며 알게 된건데 ikd-tree 기반으로 제작되었다고 해서 일단 알아보고자 논문을 읽게 되었다.

Abstract

Dynamic space partition을 위한 ikd-tree를 제안한다. ikd-Tree 는 incremental하게 k-d tree를 새로운 점들만 가지고 update 함으로서 기존 static k-d tree보다 훨씬 적은량의 연산이 요구된다. point-wise operation 외에도 robotic application에 자주 사용되는 box-wise operation이나 down-sampling도 제공한다. Incremental operation (insert, re-insert, delete)와 병행으로 ikd-Tree는 능동적으로 tree 구조를 모니터링 하며 tree를 부분적으로 balancing하여 나중에 nearest search가 효율적으로 진행되도록 한다. ikd-Tree는 세세히 engineering 되었으며 전반적인 효율을 극대화 하기 위해 multi-thread parallel computing을 지원한다. 제안하는 알고리즘의 고효율성을 나타내기 위해 이론적인 해석을 하였으며, 실험으로 무작위적인 데이터와 실제 LiDAR-inertial SLAM에서의 LiDAR point cloud에 적용하였으며 모든 테스트에서 기존 k-d tree 연산속도의 4%만 걸리는 것을 확인하였다.

Introduction

k-d Tree는 nn point를 찾는데 있어 빠른 속도를 보여 LiDAR odometry나 obstacle collision check에 사용된다. 여기서 주로 사용되는 k-d tree는 처음부터 모든 점들을 가지고 structure를 만드는 ‘static’한 방식으로 진행된다. 이는 데이터가 주로 연속적으로 들어오는 상황에 알맞지 않다. 따라서 k-d tree는 몇번에 한번씩 업데이트되거나 새로운 포인트에서만 다시 만들어지는 방식으로 진행되어왔다.

더 자연스러운 k-d tree update 방식으로는 국소적으로 업데이트 하는 방법이 될 수 있다. Local update는 전체 tree를 업데이트하는 불필요한 작업을 줄여 효과적인 연산을 수행할 수 있다. 이러한 dynamic k-d tree는 새로운 데이터가 기존 tree 보다 월씬 적을 때 매우 효과적이다.

하지만 이러한 dynamic k-d tree는 robotic application에 다양한 문제가 있다:

  1. Point operation 뿐만 아니라 down-sampling과 같은 space operation도 지원해야 한다.
  2. Tree를 쉽게 unbalance하게 만들어 애초의 목적과는 다르게 효율적인 search를 방해하기에 re-building이 요구된다.
  3. Re-building은 real-time application에 맞게 효율적이어야 한다.

이 논문에선, k-d tree를 새로운 point로만 update하며 동시에 원하는 resolution으롷 downsampling이 가능한 ikd-Tree라는 dynamic k-d tree 구조를 제시한다. 이는 insertion, re-insertion, delete와 같은 point-wise operation이나 box-wise operation을 포함한 incremnetal operation을 지원한다. Tree는 부분적인 re-building을 통해 자동으로 re-balance를 수행한다. Real-time update를 위해 ikd-Tree는 필요에 따라 tree re-building을 두개의 thread로 나눠서 진행한다. 본 논문에서는 전체 tree update에 대한 incremental operation와 re-balance에 대한 복잡도 해석을 한다. ikd-Tree의 time complexity는 random data와 LiDAR point cloud에 대해 획기적으로 줄어든 것을 확인하였다.

Related work

k-d tree는 binary search tree (BST)로 볼 수 있으며 AVL, treaps, splay trees와 같은 incermental operation을 inherit한다. 이러한 기법들에서 re-balancing은 tree node rotation을 통해 쉽게 이뤄질 수 있다. 하지만 이런 tree node rotation은 1-D 데이터에서만 가능하며, multi dimension의 경우 좀더 복잡한 reorganization이 요구된다.

k-d tree의 빠른 re-balancing에 대한 전략은 두가지로 나뉜다:

  1. 하드웨어 기반 acceleration
  2. 특별히 제작된 structure를 통한 dynamic re-balancing

하드웨어 기반 acceleration은 computing hardware를 새로운 k-d tree를 만듧으로서 re-balancing한다. (그냥 새로 만드는거 아님?) 이는 dynamic scene에서의 ray-tracing와 같은 문제를 해결하기 위해 깊게 연구된 바 있다. 3D Graphics에서 CPU, multi-core CPU, GPU에 대한 연구가 진행된 바 있으나, computing power에 의존하여 onboard computer에선 한계가 있다.

다음은 후자에 대한 내용이다:

  • Bentley et at.은 static k-d tree를 dynamic k-d tree로 변환하는 방법에 대해 제안한 바 있다. 이 dynamic k-d tree는 insertion만 지원하고 delet를 지원하지 않아 nearest search 시간이 tree가 증가함에 따라 증가한다.
  • Galperin et al.은 scapegoat k-d tree를 제안하였으며, 이는 unbalance된 sub-tree를 re-building함으로써 re-balancing하는 알고리즘이다.
  • Bkd Ttree는 외부 memory 사용에 집중한 k-D-B tree를 확장한 dynamic data structure이다. static k-d tree set들 내에서 re-balance와 re-building이 특정 구간에서 이루어지는 방식이다.
  • 잘 알려진 PCL은 FLANN search를 사용한다. Point insertion과 delete는 FLANN에서 제공되나, re-balancing은 일정량의 insertion과 delete 이후 모든 tree를 rebuilding하는 매우 비효율적인 방법을 사용한다.

ikd-Tree는 효율적으로 incremental operation과 dynamic re-balacing을 수행한다. Bentley et al. 과 비교했을 때 ikd-Tree는 delete를 지원한다. 기존 연구와 비교했을 때 ikd-Tree는 point, box의 incremental operation을 지원하며, 동시에 point down sampling도 지원한다. ikd-Tree의 dynamic tree re-balancing 기법은 unbalance된 sub tree만 rebuild하는 scapegoat tree 기법을 따른다. ikd-Tree는 특히 real-time LiDAR mapping이나 motion planning과 같이 데이터가 연속적으로 들어오고 빠른 incremental update가 요구되는 robotic application에 유용하다.

ikd-Tree design and implementation

Data structure

2~4번쨰 줄은 기존 k-d tree에 사용되는 attribute들이다. leftson과 rightson은 좌우 자식 노드에 대한 pointer이며 point coordinate, intensity와 같은 값들이 point에 저장된다. Point 하나가 node 하나에 대응됨으로 이를 혼용하여 사용한다. Division axis는 axis에 저장된다. 5~7번째 줄이 ikd-Tree에서 새로 사용하는 attribute들이다.

Building an incremental k-d tree

Incremental k-d tree를 만드는 과정은 static k-d tree를 만드는 과정과 비슷하나, incremental update를 위해 추가적인 정보를 유지하는 차이가 있다.

주어진 point array V에 대해서 point들은 먼저 maximal covarinace를 가진 division axis로 나뉘다. (4~5번째 줄) (이 covariance 뭔뜻임?) 그 후 median point가 새로운 tree node T에 저장된다. (6~8번째 줄). Medain point의 상하에 있는 점들은 T의 left and right son node로 전달된다 (9~10번째 줄). 11~12번째 줄의 lazylabelinit과 pullup은 Data structure의 5~7번째의 attribute들을 update한다.
